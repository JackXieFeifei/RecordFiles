使用说明：
以下为开发过程中测试用包打包注意事项，如无需求请无视：
首先，修改local.properties文件内容为自己的路径
之后，确认danji文件夹下的build.py文件中的变量PEIZHI_STR（第18行）所定义的配置文件名称为“build_config.json”；
Python build.py -c thranQA  打出thran渠道测试环境的包
Python build.py -c thranBeta  打出thran渠道仿真环境的包
Python build.py -c thranRelease  打出thran渠道生产环境的包

Python build.py -c ceshiQA  打出ceshi渠道测试环境的包
Python build.py -c ceshiBeta  打出ceshi渠道仿真环境的包
Python build.py -c ceshiRelease  打出ceshi渠道生产环境的包

Python build.py -c yyb1QA  打出yyb1渠道测试环境的包
Python build.py -c yyb1Beta  打出yyb1渠道仿真环境的包
Python build.py -c yyb1Release  打出yyb1渠道生产环境的包

Python build.py -c ceshiKF31  打出ceshi渠道开发环境的包 ip为172.28.14.31  CommonSocket端口号为22347   PositionSocket端口号为22350
Python build.py -c ceshiKF34  打出ceshi渠道开发环境的包 ip为172.28.14.34  CommonSocket端口号为22347   PositionSocket端口号为22350

    特别注意：打包完成后，将手动修改的地方回复原状，回复原状，回复原状！！！！！


newchuan五个包，在打包前请把manifest文件夹中的newchuanManifest.xml内的<application>标签下内容拷贝至
我们自己的AndroidManifest.xml文件夹下的<application>标签下
（因未知原因，使用打包工具打该包时，会出现重复拷贝的问题）
    特别注意：打包完成后，将手动修改的地方回复原状，回复原状，回复原状！！！！

然后使用Python build.py打出其他的包

huawei及360的包  去掉其他支付方式，对方会检测(已在打包配置文件中配置，无需操作)
